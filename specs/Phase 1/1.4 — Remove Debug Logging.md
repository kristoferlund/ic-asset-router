# 1.4 — Remove Debug Logging

**Phase:** 1 — Fix Foundations
**Priority:** Minor
**Status:** Not started

## Problem

`ic_cdk::println!` calls exist on every request path in `src/lib.rs` and `src/router.rs`. These print to the canister's debug log on every single HTTP request, adding unnecessary overhead in production.

## Objective

Gate all debug logging behind a Cargo feature flag so it's opt-in during development and completely compiled out in production builds.

## Specification

### Feature Flag

In `Cargo.toml`:
```toml
[features]
default = []
debug-logging = []
```

### Conditional Compilation

Replace all `ic_cdk::println!` calls with a macro:

```rust
#[cfg(feature = "debug-logging")]
macro_rules! debug_log {
    ($($arg:tt)*) => { ic_cdk::println!($($arg)*) };
}

#[cfg(not(feature = "debug-logging"))]
macro_rules! debug_log {
    ($($arg:tt)*) => {};
}
```

Then replace all occurrences:
```rust
// Before:
ic_cdk::println!("Matched route: {}", path);

// After:
debug_log!("Matched route: {}", path);
```

## Files to Modify

- `Cargo.toml` — Add feature flag
- `src/lib.rs` — Replace `ic_cdk::println!` with `debug_log!`
- `src/router.rs` — Same
- New: `src/macros.rs` or inline in `src/lib.rs` — Define the `debug_log!` macro

## Acceptance Criteria

- [ ] Zero `ic_cdk::println!` calls remain outside the macro
- [ ] Default build (no features) produces no debug output
- [ ] `cargo build --features debug-logging` restores all debug output
- [ ] No performance overhead from logging in default builds (compiled out, not runtime-checked)
