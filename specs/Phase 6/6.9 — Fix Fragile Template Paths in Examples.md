# 6.9 — Fix Fragile Template Paths in Examples

## Problem

The `tera-basic` example uses a fragile relative path for template inclusion:

```rust
include_str!("../../../../templates/post.html")
```

This path depends on the exact directory nesting depth (`src/routes/posts/_postId/index.rs` → 4 levels up to reach `templates/`). If the route is moved to a different depth, the path breaks silently at compile time.

## Specification

Replace fragile relative `include_str!` paths with `CARGO_MANIFEST_DIR`-anchored absolute paths:

```rust
include_str!(concat!(env!("CARGO_MANIFEST_DIR"), "/templates/post.html"))
```

`CARGO_MANIFEST_DIR` is set by Cargo during compilation and points to the crate root. This works in all source files regardless of their nesting depth.

### Files to modify

| File | Change |
|------|--------|
| `examples/tera-basic/src/routes/posts/_postId/index.rs` | Replace `include_str!("../../../../templates/post.html")` with `include_str!(concat!(env!("CARGO_MANIFEST_DIR"), "/templates/post.html"))` |

Check other examples for the same pattern and fix if found.

### Acceptance criteria

- [ ] No relative `../` paths remain in `include_str!` calls in any example
- [ ] `cargo check --manifest-path examples/tera-basic/Cargo.toml` succeeds
- [ ] `cargo check` on all other examples still succeeds
